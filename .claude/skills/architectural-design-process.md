# Architectural Design Process Skill

How a commercial architectural project moves from "we should build a
thing" to operational facility. Apply to project planning, fee
proposals, phase sequencing, owner conversations, and reading another
firm's deliverables.

---

## Phase overview (AIA conventions)

| Phase | Owner action | Architect deliverable | Cost certainty |
|---|---|---|---|
| **Pre-design / Programming** | Hire architect + state needs | Program; site analysis; budget ROM | ±30% |
| **Schematic Design (SD)** | Approve direction | Floor plans, key elevations, building section, code analysis | ±15% |
| **Design Development (DD)** | Approve detail; commit budget | Refined drawings; selected systems; specs outline | ±10% |
| **Construction Documents (CD)** | Approve to permit + bid | Stamped drawings + specs | ±5% |
| **Bidding / Negotiation (B/N)** | Select contractor | Addenda; bid analysis | bid number |
| **Construction Administration (CA)** | Pay contractor | Submittal review; RFIs; site visits; pay app review | actual |
| **Closeout** | Accept project | Punch list; as-built support | actual |

## Programming

### Goal
Establish the right thing to build before drawing anything.

### Activities
- Owner interviews (decision-makers + end users).
- Functional program: spaces, NSF, departmental adjacencies, growth.
- Operational program: how the building runs day-to-day.
- Site analysis: zoning, easements, utilities, topo, soil.
- Code analysis: zoning + occupancy class + envelope code.
- Budget ROM: $/sf for this typology in this market + escalation.
- Schedule ROM.

### Deliverables
- **Program document** (typically a Word doc + Excel space schedule).
- **Site analysis report**.
- **Budget ROM letter**.
- **Schedule ROM Gantt chart**.

### Owner sign-off
The owner signs the program. This is the contract for what's being
designed.

---

## Schematic Design (SD)

### Goal
Test that the program fits the site at a buildable concept.

### Activities
- Site plan: building footprint, parking, circulation, utilities.
- Floor plans: program-compliant; major dimensions; structural grid.
- Building sections (1-2): floor-to-floor heights, roof slopes.
- Key elevations: massing, fenestration concept.
- Code analysis sketch: use group(s), construction type, occupant load,
  egress concept, fire-rated separations.
- Outline of structural system + mechanical system.
- 3D massing / rendering for owner buy-in.

### Deliverables
- **SD set**: 10-20 sheets typical for mid-sized commercial.
- **Design narrative** (2-5 pages): what the building is + why.
- **Outline specs** (one paragraph per division).
- **Updated cost estimate**.

### Owner sign-off
SD approval locks the design direction. Changes after this point cost
money.

---

## Design Development (DD)

### Goal
Develop the building enough to coordinate disciplines + price.

### Activities
- Drawing development: plans, elevations, sections drawn to
  intermediate detail.
- Wall types, door types, window types defined.
- Major details started (typical wall section, jamb, sill, head).
- Finishes selected (primary visible materials).
- Structural sized: column sizes, beam depths, slab thicknesses.
- MEP routing studied: shafts, mechanical room, plenums.
- Cost estimate updated; owner reviews.
- Constructability review (if CMAR is on board).

### Deliverables
- **DD set**: 25-40 sheets for mid-sized commercial.
- **Specifications outline**: list of sections; brief one-paragraph each.
- **Coordinated discipline drawings**: arch / struct / MEP all on the
  same level + grid.
- **Updated cost estimate**.

### Owner sign-off
DD approval is the "commit" gate. After this, scope is frozen unless
the owner agrees to a change order.

---

## Construction Documents (CD)

### Goal
Document the building completely enough to permit, bid, and build.

### Activities
- All drawings completed to permit + bid level.
- All schedules complete (door, window, finish, equipment, hardware).
- All details developed (typical + unique conditions).
- All specifications written (3-part format; Divisions 00-49).
- Code analysis sheets stamped.
- Consultant drawings coordinated (clash-detected if BIM).
- Architect stamps the set.

### Deliverables
- **CD set**: 60-150 sheets for mid-sized commercial.
- **Project manual** (specifications, bound separately).
- **Permit submission package** per jurisdiction.
- **Final cost estimate** (or GMP if CMAR).

### Owner sign-off
CD approval = ready to bid + permit. Changes from here become COs.

---

## Bidding / Negotiation (B/N)

### DBB process
- Issue for bid: distribute documents to bidders.
- Pre-bid meeting + site walk.
- Bidder RFIs → architect issues addenda.
- Bid opening: usually public for government work.
- Bid analysis: recommend contractor to owner.

### CMAR process
- CM has been on the team since SD/DD.
- GMP (Guaranteed Maximum Price) is negotiated.
- Subs are bid by the CM; architect reviews bid packages.

### Deliverables
- **Addenda** (Addendum 1, 2, 3…).
- **Bid analysis / contractor recommendation letter**.

---

## Construction Administration (CA)

### Goal
Get the building built per the documents.

### Activities
- **Submittals**: review shop drawings, product data, samples within
  contract turnaround (typically 14 days).
- **RFIs**: respond within contract turnaround (typically 5-10 days).
- **ASIs**: Architect's Supplemental Instructions for clarifications.
- **Change Orders**: scope/cost/time changes requiring owner approval.
- **Pay applications**: review monthly + certify to owner.
- **Site visits**: typically bi-weekly; field reports.
- **Mockups**: review + approve before production.
- **Substantial completion**: certify + start punch list.
- **Final completion**: close out + certificate of occupancy.

### Deliverables
- **Submittal log** (digital, often in Procore or Bluebeam).
- **RFI log**.
- **Change Order log**.
- **Field reports** (one per site visit).
- **Punch list**.
- **Certificate of Substantial Completion**.
- **Certificate of Final Completion**.

---

## Closeout

### Goal
Hand the building over and start the warranty.

### Activities
- Contractor delivers as-built drawings + O&M manuals + warranties.
- Architect reviews + accepts.
- Owner training: BMS, security, FM systems.
- 11-month walk-through (before 1-year warranty ends): identify warranty
  items.

### Deliverables (owner receives)
- **As-built drawings** (PDF + DWG).
- **Project manual + specs** (final).
- **O&M manuals** (paper + digital).
- **Warranties** (each system).
- **Closeout binder / digital archive**.

## AIA contract documents (common ones)

| Document | Use |
|---|---|
| **B101** | Owner ↔ Architect agreement (most common; standard form) |
| **A101** | Owner ↔ Contractor stipulated sum |
| **A102** | Owner ↔ Contractor CM at Risk + GMP |
| **A201** | General Conditions of the Contract (companion to A101/A102) |
| **C401** | Architect ↔ Consultant (structural, MEP) |
| **G701** | Change Order form |
| **G702/703** | Application + Certificate for Payment (pay app) |
| **G704** | Certificate of Substantial Completion |

The AIA contract is the framework. Don't draft from scratch; use the
form + modifications.

## Fee + scope structure (typical)

### Fee basis
- **% of construction cost**: 5-10% typical for commercial new
  construction; 12-15% for renovation.
- **Lump sum**: when scope + schedule are well-defined.
- **Hourly + max**: for studies, programming, early work.

### Fee distribution by phase (rough)
- Pre-design / Programming: 5-10% of fee
- Schematic Design: 15%
- Design Development: 20%
- Construction Documents: 40%
- Bidding / Negotiation: 5%
- Construction Administration: 20%

## Owner approval gates (don't skip)

```
Program approved? → no design.
SD approved? → no DD.
DD approved? → no CD.
CD approved? → no permit.
Permit issued? → no construction start.
SC certified? → no occupancy.
```

Document every approval in writing. The day someone says "I never
approved that" is the day you wish you had the email.

## When to invoke

- Writing a fee proposal
- Sequencing project phases
- Setting owner expectations on deliverables
- Reviewing another firm's deliverables for completeness
- Coordinating with consultants on phase timing
- Planning RFI / submittal turnaround during CA
- Closeout planning
